# Requisitos do Frontend — Controlei

## Requisitos Funcionais (RF)

### RF01 - Cadastro de usuário
O sistema deve permitir que o usuário crie uma conta informando:

- nome
- email
- senha

### RF02 - Login
O sistema deve permitir autenticação de usuários cadastrados.

### RF03 - Recuperação de senha
O sistema deve permitir que o usuário solicite recuperação de senha.

### RF04 - Registro de transações financeiras
O usuário deve poder registrar:

- Receitas
- Despesas
- Investimentos

### RF05 - Visualização de transações
O usuário deve visualizar suas transações registradas.

### RF06 - Dashboard financeiro
O sistema deve apresentar um dashboard contendo:

- resumo financeiro
- gráficos
- metas financeiras

### RF07 - Gerenciamento de metas
O usuário deve poder:

- criar metas
- visualizar metas
- acompanhar progresso

### RF08 - Atualização de dados
O sistema deve permitir edição de transações.

### RF09 - Exclusão de registros
O sistema deve permitir excluir transações.

---

# Requisitos Não Funcionais (RNF)

### RNF01 - Performance
O sistema deve carregar páginas em menos de **2 segundos**.

### RNF02 - Responsividade
O sistema deve funcionar em:

- desktop
- tablet
- mobile

### RNF03 - Segurança
Dados sensíveis devem ser protegidos através de autenticação segura.

### RNF04 - Usabilidade
A interface deve ser simples e intuitiva.

### RNF05 - Manutenibilidade
O código deve seguir boas práticas de organização e separação de responsabilidades.
